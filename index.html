Hierarchical Streaming Memory with Mamba State Compression for Continuous Speech Separation

Submitted as conference paper at National Conference on Man-Machine Speech Communication, NCMMSC 2026

1Yanlin Wang, 1,2Ying Hu

1 School of Computer Science and Technology, Xinjiang University, Urumqi, China
2 Key Laboratory of Signal Detection and Processing, Xinjiang, Urumqi, China


Abstract

Continuous speech separation (CSS) extracts individual speaker streams from unbounded audio mixtures under streaming constraints. Existing streaming methods either grow memory cost linearly through attention-based key-value caching or employ single-vector compression that loses speaker-specific information. We propose HSMnet, a lightweight streaming CSS framework centered on Hierarchical Streaming Memory (HSM) — a three-tier architecture that compresses unbounded streaming history into M constant-size memory slots via unidirectional Mamba3 state compression with EMA stabilization, achieving approximately 200× compression per chunk. HSM employs per-band cross-attention fusion, enabling each frequency band to independently retrieve relevant historical speaker identity from memory slots without growing computational cost. Speaker tracking is handled by a two-layer Transformer Dual-Source Attractor (DSA) with time-sampled key-value tokens and causal masked self-attention for explicit speaker competition. The complete pipeline integrates a Mamba-based spectral encoder, parameter-sharing time-frequency interaction blocks, and a Transformer-based dual-source attractor for speaker tracking, together with the proposed hierarchical memory and attractor mechanisms, into a compact 5.8M-parameter system achieving sub-second streaming latency. Across five benchmark datasets, HSMnet achieves state-of-the-art performance on EchoSet and single-channel LibriCSS, while remaining competitive with full-attention baselines on WSJ0-2Mix, Libri2Mix, and LRS2-2Mix, all while maintaining O(1) memory per chunk.

Introduction

Real-time overlapping speech separation is essential for live meeting transcription, hearable devices, and augmented reality assistants. Offline models from deep clustering and PIT through Conv-TasNet, DPRNN, SepFormer, and TF-GridNet leverage full-sequence bidirectional context to achieve state-of-the-art performance. However, these assumptions are incompatible with the latency constraints of continuous speech separation (CSS), which requires processing unbounded audio streams in real time.

Streaming approaches address latency through chunk-wise causal processing with limited look-ahead. Under such constraints, separation quality becomes critically dependent on the system's ability to leverage historical context — yet conventional attention-based key-value caching incurs growing computational overhead proportional to stream duration and fails to enforce consistent speaker identity across chunk boundaries, causing permutation errors that degrade downstream ASR.

The fundamental challenge is therefore: How can unbounded streaming history be summarized into a fixed-size state that preserves speaker identity and interaction cues while supporting efficient online propagation?

We propose HSMnet, a streaming CSS framework addressing this challenge through three contributions:

  1. Hierarchical Streaming Memory (HSM) — a three-tier architecture combining unidirectional Mamba3 state compression with Exponential Moving Average (EMA) stabilization and per-band cross-attention fusion. Inspired by Papez's Auditory Multi-slot Learnable Working Memory and Emformer's context compression, HSM replaces attention caches with M constant-size memory slots achieving ~200× compression while preserving frequency-resolved speaker identity.
  2. Two-layer Dual-Source Attractor (DSA) with structured key-value tokens — extending SepTDA and A-DCSS's null attractor with (i) time-sampled dual-source KV construction, (ii) DETR-style two-layer decoder with causal masked self-attention for explicit speaker competition, and (iii) existence prediction for automatic speaker counting.
  3. Compact constant-memory streaming CSS framework — combining efficient spectral modeling, hierarchical memory propagation, and attractor-based speaker tracking into a unified architecture with only 5.8M parameters, configurable latency from 800ms to 2000ms, and O(1) memory complexity per chunk.
